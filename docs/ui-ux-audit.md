# AutoQC UI/UX Audit Tracker

Canonical tracker for the full UI/UX audit, implementation, and verification loop requested for the AutoQC workspace.

## Review passes

- Visual/layout review: app shell, left rail, drawing viewer, right rail, responsive breakpoints, long labels, dense controls.
- Workflow/usability review: upload/sample project, manual ChatGPT/Copilot bridge, AI JSON preview/import, findings review, inspector edits, export.
- Accessibility review: keyboard focus, disabled states, labels, aria status/error feedback, link/button affordances.
- Error/edge-state review: empty projects, no sheets, no findings, filtered no-match, malformed AI JSON, backend unavailable, failed export, no accepted findings.
- Test/regression review: TypeScript build path, backend pytest suite, smoke-test feasibility, code-level route/workflow review.

## Remaining known issues / constraints

- Direct live AI-provider behavior was not changed or tested; the preserved primary product direction remains upload PDF/sample -> Chat Prompt -> paste AI update JSON -> preview/import findings -> review/edit findings -> export marked PDF.
- The tab controls are reachable and visible by keyboard, but they do not yet implement full arrow-key tablist behavior.
